TUDO SOBRE O 'CACHE'

O que é o Cache e por que ele é importante

O termo inglês "caching" significa "guardar algo (normalmente em segredo) para usar mais tarde", e é exatamente essa a função do Cache em seu navegador.


free website templates
Quando visitamos qualquer website, o navegador pega "pedaços" desse site (por exemplo Imagens, logos, fundos, parte do código HTML, arquivos CSS e JavaScript) e faz o armazenamento em nosso computador ou dispositivo de acesso.
O navegador faz o gerenciamento desses pedaços e armazena arquivos conhecidos como "acessos estáticos", ou seja, elementos que não são de comum modificação no site, como o Logo por exemplo, ou uma imagem de fundo. Elementos que são comuns para todos os usuários e que não se modificam de acesso para acesso.
O que guardar e por quanto tempo guardar é determinado pelo site ou portal. Alguns arquivos são removidos poucos dias depois, enquanto outro podem durar até um ano!
Aqui vem a parte onde acompanha um quesionamento: Mas se o site está armazenando arquivos, como fica minha segurança?
Os benefícios do cache do navegador superam os riscos. Bons firewalls, antivírus e bom senso são tudo o que você precisa para manter sua máquina segura.

Mas quais são os benefícios do armazenamento em Cache?

Em resumo, o benefício do Cache é a diminuição do tempo de carregamento do site.

Quando acessamos pela primeira vez uma site, seu navegador tem uma conversa com o Server remoto aonde os arquivos do site estão armazenados. O navegador faz pedidos, requisições, e o Servidor envia as respostas, tratando aquelas requisições e trazendo os arquivos necessários.

A estrutura HTML é a primeira a ser baixada, e é toda a estrutura do site. O navegador consegue "traduzir" essa linguagem, o que gera mais requisições ao servidor, para mais "pedaços" da página.

Esse processo leva um tempo, e consome um pouco da sua largura de banda, ou pacotes de internet no caso do acesso móvel.
 
Textos são carregados mais rápidos que os outros elementos (como imagem, formatação, posicionamento, etc). Isso porque um pedaço de texto é muito menor e mais rápido para ser carregado do que uma imagem em alta resolução, que pode levar alguns segundos para ser carregada completamente (o que é uma eternidade em tempo computacional).


Aí entra o Cache!

O trabalho do Cache é justamente armazenar esses elementos para que, num acesso futuro, eles já estejam "pré carregados" no seu Disco Rígido, e seu acesso seja mais rápido e fluído, economizando tempo computacional e, largura de banda e, no caso de um acesso móvel, pacotes de internet.

Legal, não é? ;)

Se ter arquivos Cache na minha máquina é tão bom, por que as vezes eu preciso limpá-los?

O maior benefício do cache também é seu pior desafio!

Imagine o seguinte: Você acessa hoje um site e os arquivos cache ficam armazenados no seu computador. Porém, durante a noite, uma atualização foi lançada, modificando um logo, uma imagem ou até mesmo um elemento estrutural da página.
No dia seguinte, quando acessar aquele mesmo site, a atualização parece não surtir efeito no seu navegador, e os novos elementos não aparecem.

Isso ocorre porque você está com o Cache desatualizado. E isso pode causar alguns problemas como:

1

IMAGENS DESATUALIZADAS

Cache antigo pode gerar o carregamento de imagens desatualizadas. O que não é bom nem para o usuário, nem para o site.

2

ARQUIVOS DE ESTRUTURA PODEM NÃO SER CARREGADOS DE FORMA CORRETA

Caso o seu cache fique muito tempo sem um refresh, a estrutura do site visualizado pode não ser a mais atual, ou corrigida. Por exemplo de uma novo parágrafo de texto foi adicionado, e seu cache continua desatualizado, poderá gerar uma visualização não precisa daquele texto como um todo.                                 

3

FUNÇÕES E PARTES DO CÓDIGO PODEM NÃO FUNCIONAR

Imagine que uma nova função para o site foi criada recentemente, onde usa um novo elemento criado para criar um redirecionamento. Se uma nova função JavaScript, elemento de CSS ou alguma estrutura HTML não for devidamente carregada, essa função apresentará um erro no Console, e a experiência do usuário estará comprometida
